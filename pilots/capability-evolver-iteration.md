## Step 3: 评分表定义

### 评分表设计原则

| 原则 | 实施 |
|------|------|
| **二元检查项** | yes/no，非模糊判断 |
| **3-6 个检查项** | 避免过多导致"刷题" |
| **权重可选** | 核心项可加权 |

### 评分表 (5 个检查项)

```markdown
## 评分表：capability-evolver

### 检查项 (5 个)

1. 是否识别了至少 3 个具体问题
   - 是 = 1 分，否 = 0 分
   - 权重：2 (核心项)

2. 每个问题是否有具体改进建议
   - 是 = 1 分，否 = 0 分
   - 权重：2 (核心项)

3. 是否提供了可执行的代码/示例
   - 是 = 1 分，否 = 0 分
   - 权重：1

4. 是否追踪了进化历史 (与上一版对比)
   - 是 = 1 分，否 = 0 分
   - 权重：1

5. 是否避免了泛化建议 (如"优化代码"而无具体内容)
   - 是 = 1 分，否 = 0 分
   - 权重：1

### 总分计算
总分 = Σ(检查项得分 × 权重) / Σ权重 × 100%
总分 = (加权得分) / 7 × 100%
```

### 评分表验证

- [ ] 所有检查项是 yes/no 二元
- [ ] 检查项数量 5 个 (在 3-6 范围内)
- [ ] 核心项已加权 (问题识别 + 改进建议)
- [ ] 评分标准明确无歧义
